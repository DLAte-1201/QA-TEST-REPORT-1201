# 개인 포트폴리오 웹사이트 품질 검증 테스트 리포트

## 1. 테스트 개요

본 문서는 개인 포트폴리오 정적 웹사이트의 기본 기능, 화면 표시, 반응형 레이아웃, 접근성, 사용성, 콘텐츠 일관성을 검토한 품질 검증 테스트 리포트이다.

테스트 대상은 GitHub Pages에 배포된 라이브 페이지와 현재 저장소의 정적 파일으로, 라이브 페이지는 `https://dlate-1201.github.io/` 기준으로 주요 콘텐츠와 링크 구조를 확인했으며, 세부 구현은 `index.html`, `styles.css`, `script.js`, `README.md`를 기준으로 정적 검토했다.

## 10. 회고

- 이번 테스트에서는 개인 포트폴리오 웹사이트의 주요 기능이 정상적으로 작동하는지 확인하는 데 중점을 두었다. 특히 화면 표시, 주요 링크 이동, 반응형 레이아웃, 정보 노출, 콘텐츠 일관성 측면에서 기능상 문제가 없는지 검증했다.
- 예상보다 잘 되어 있었던 부분은 주요 테스트 케이스에서 Fail 항목이 확인되지 않았다는 점이다. 이를 통해 현재 홈페이지가 심각한 기능적 하자 없이 기본적인 사용 흐름을 제공하고 있음을 확인할 수 있었다.
- 발견한 문제 또는 개선점으로는 모바일 환경에서의 보조 닫기 동작이 부족한 부분, 메인 제목이 포트폴리오의 목적을 충분히 강하게 드러내지 못하는 부분 등이 있었다. 치명적인 결함은 아니지만, 사용 편의성, 시인성, 전달력을 높이기 위해 개선할 수 있는 항목으로 정리했다.
- QA 관점에서 배운 점은 AI 도구를 활용하면 테스트 관점 도출, 코드 검토, 리포트 초안 작성 등의 작업을 효율적으로 진행할 수 있지만, 모든 검증을 AI에 의존할 수는 없다는 점이다. 특히 기업 내부 데이터, 개인정보, 보안상 민감한 정보가 포함된 환경에서는 AI 도구 사용이 제한될 수 있으므로, AI 시대에도 사용자의 직접 검증 능력과 기본적인 QA 역량은 여전히 중요하다는 점을 확인했다.
- 다음 프로젝트에서는 의도적으로 결함을 삽입한 웹페이지 또는 작업물을 만들고, Codex가 해당 결함을 얼마나 효과적으로 발견하며 테스트 케이스와 결함 기록으로 정리할 수 있는지 확인해보고 싶다. 이를 통해 AI 보조 QA의 가능성과 한계를 더 구체적으로 검증할 계획이다.
